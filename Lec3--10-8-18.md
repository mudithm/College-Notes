<li><code>M-q</code> to autowrap individual paragraphs</li>
</ul>
</li>
</ul>
</li>
</ul>

<h4><a id="user-content-carriage-returns" class="anchor" aria-hidden="true" href="#carriage-returns"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Carriage Returns</h4>
<ul>
<li>In Linux, <code>\n</code> represents new line</li>
<li>In windows, this will be <code>\r\n</code>, or a carriage return followed by a line feed</li>
<li>These can cause some corruption of files between OS, so make sure to remove them</li>
</ul>

<h4><a id="user-content-environment-variables" class="anchor" aria-hidden="true" href="#environment-variables"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Environment Variables</h4>
<ul>
<li>Can be accessed from any child process</li>
<li>Common ones:
<ul>
<li><code>HOME</code>: path to the home directory</li>
<li><code>PATH</code>: list of directories to search in for a command to execute</li>
<li>Change value:
<ul>
<li><code>export &lt;VARIABLE&gt;=...</code></li>
</ul>
</li>
</ul>
</li>
<li><code>LC</code> environment variables:
<ul>
<li>Ex: <code>LC_TIME</code> for date and time, <code>LC_NUMERIC</code> for non-monetary numeric formats, <code>LC_COLLATE</code> for comparing and sorting</li>
<li>These variables give information to the <code>locale</code> command</li>
<li><code>LC_COLLATE</code> Examples:
<ul>
<li><code>LC_COLLATE='c'</code>: sorting in ASCII order</li>
<li><code>LC_COLLATE='en_US'</code>: sorts in case insensitive, except when the two strings are otherwise equal and one has an uppercase letter earlier than the other</li>
</ul>
</li>
</ul>
</li>
</ul>

<h4><a id="user-content-the-c-locale" class="anchor" aria-hidden="true" href="#the-c-locale"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>The 'C' Locale</h4>
<ul>
<li>Default locale</li>
<li>environment of "least surprise"</li>
</ul>
<hr>
<h4><a id="user-content-sort-comm-and-tr" class="anchor" aria-hidden="true" href="#sort-comm-and-tr"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a><code>sort</code>, <code>comm</code>, and <code>tr</code></h4>
<ul>
<li><code>sort</code>: sorts lines of text files
<ul>
<li>Usage: <code>sort &lt;options&gt; ... &lt;file&gt; ...</code></li>
<li>Sort order depends on the locale</li>
<li>C locale: ascii sorting</li>
</ul>
</li>
<li><code>comm</code>: compare two sorted files line by line
<ul>
<li>Usage: <code>comm &lt;options&gt; ... file file2</code>
<ul>
<li>requires 2 files, must be sorted according to <code>LC_COLLATE</code></li>
</ul>
</li>
</ul>
</li>
<li><code>tr</code>: translate or delete characters
<ul>
<li>Usage: <code>tr &lt;option&gt; ... SET1 &lt;SET2&gt;</code></li>
<li><code>cat file1 | tr 'a-cm-o' '123abc</code>
<ul>
<li>translates a to 1, b to 2, c to 3, and m to a, n to b, and 0 to c</li>
</ul>
</li>
<li><code>cat file2 | tr -d 'akm' &gt; file1</code>
<ul>
<li>removes all a's, k's, and m's in file2 and puts the result in file1</li>
</ul>
</li>
</ul>
</li>
</ul>
<blockquote>
<p>Note about pipelining: you cannot normally read and later write to the same file in one command.</p>
</blockquote>

<h2><a id="user-content-scripting-and-the-shell" class="anchor" aria-hidden="true" href="#scripting-and-the-shell"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Scripting and the Shell</h2>
<hr>
<h4><a id="user-content-the-shell-and-the-os" class="anchor" aria-hidden="true" href="#the-shell-and-the-os"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>The Shell and the OS</h4>
<ul>
<li>shell: user interface to OS</li>
<li>Accepts commands as text, uses OS API to carry them out</li>
<li>Common shells: <code>bash</code>, <code>csh</code>, <code>ksh</code></li>
<li><code>Bash</code> is an interpreted language, which means it is portable but much slower to execute than a compiled language like <code>C++</code></li>
</ul>

<h4><a id="user-content-scripts-first-line" class="anchor" aria-hidden="true" href="#scripts-first-line"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>Scripts: First Line</h4>
<ul>
<li>Shell script file is a file with shell commands</li>
<li>When shell script is executed,
<ul>

<li>first line is used to state which child shell to use
<ul>
<li><code>#! /bin/sh</code></li>
<li><code>sh</code> is a symbolic link to the current shell, usually <code>bash</code></li>
<li><code>#! /bin/bash</code>, etc.</li>
</ul>
</li>
<li>Tells the computer which interpreter to use</li>
</ul>
</li>
</ul>
